# Stretch — Demo script (happy path + failure path)

| Field | Value |
|-------|-------|
| **Priority** | Stretch |
| **Status** | Open |
| **Area** | Documentation / Demo Day |

## Problem

Demo Day and reviewer walkthroughs go smoother with a **short script**: commands, URLs, one success flow, one deliberate failure (e.g. validation, dispatch conflict, downstream error).

## Value

Stronger **communication** score and less ad-hoc live debugging.

## Acceptance criteria

- [ ] `SUBMISSION.md` or `docs/` contains a numbered script under 15 minutes.
- [ ] Failure path demonstrates observable error handling (UI or API).
